# Vendor Risk Policy

## Risk tiering

The procurement owner should assign a preliminary risk tier during triage.

### Low risk

Examples:
- Office supplies
- Facilities vendors without system access
- Event vendors without employee or customer data access
- Annual contract value below $25,000
- No confidential data processing

### Medium risk

Examples:
- SaaS tools processing business contact information
- Vendors with limited CRM or collaboration-tool access
- Vendors with standard contract terms and current security attestation
- Annual contract value between $25,000 and $100,000

### High risk

Examples:
- Vendors processing customer PII
- Vendors processing employee sensitive data
- Vendors connecting to HRIS, finance, data warehouse, identity, or production systems
- AI vendors using company, customer, or employee data
- Vendors with subprocessors outside the United States
- Vendors with incomplete security materials
- Annual contract value above $100,000

## Executive approval

Executive approval is required when:

- A vendor uses customer or employee data for AI model training
- A high-risk vendor cannot provide a current SOC 2 Type II report or equivalent
- A vendor processes sensitive employee data
- Total contract value exceeds $250,000
- The vendor is business-critical and lacks standard security controls

## Duplicate vendor check

Procurement should check whether a vendor already exists under a similar name before creating a new vendor record.

If a likely duplicate is found, the agent should flag the record for human review rather than creating a new-vendor recommendation.
