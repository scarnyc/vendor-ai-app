# Data Handling Policy

## Data classes

### Public data

Information intended for public release.

### Internal data

Operational information that is not public but does not contain personal data, sensitive data, or confidential commercial information.

### Confidential data

Information that could harm the company, customers, employees, or partners if disclosed. Examples include:

- CRM opportunity history
- Sales activity notes
- Customer workflow metadata
- Internal financial information
- Vendor pricing
- Product roadmap information

### Restricted data

Highly sensitive information requiring strict review. Examples include:

- Customer personal information
- Employee personal information
- Employee performance or compensation data
- Authentication credentials
- Production data
- Financial account data
- Regulated or sensitive personal data

## AI and model training

Company, customer, and employee data may not be used for vendor model training, benchmarking, or product improvement unless explicitly approved by Legal, Security, and an executive sponsor.

Ambiguous language such as "service improvement," "model enhancement," "benchmarking," or "product analytics" should be flagged for Legal and Security review when personal data or confidential data is involved.

## Cross-border processing

If personal data or confidential data is processed by subprocessors outside the United States, Legal and Security review is required.
